### T002: Write Visual Layer Principles (P1-P4)

<action>
Write 4 visual layer principles in `.planning/design-philosophy.md`, after the introduction section.

Each principle follows the format:
  ### P{N}: {7-word title}
  **{Core statement — one sentence}**
  
  **Why:** {2-3 sentences rationale}
  
  **DO:** {Concrete text-described example}
  **DON'T:** {Concrete text-described failure}

P1: 暖色调优先 (Warm Colors First)
- Statement: Cream, amber, and warm gray form the foundation — avoid cold blues and pure black/white.
- Rationale: Pure white reads as clinical; pure black feels hostile. Warm tones mirror sunlight and natural materials.
- DO: Page background #faf8f5, text #37352F, accent #e8923a
- DON'T: Pure #fff background, blue/purple link colors, pure #000 text

P2: 留白是结构 (Whitespace as Structure)
- Statement: Space communicates hierarchy — generous margins replace boxes, borders, and dividers.
- Rationale: Borders and shadows add visual noise. Whitespace is the quietest way to say "this belongs together, that doesn't."
- DO: 24px section gaps, 8px within components, single rule line only for content separation
- DON'T: Boxed-in sections with visible borders, 2px divider lines everywhere, cramped layouts

P3: 圆润亲和 (Rounded Affinity)
- Statement: Soft corners feel approachable — use 6-16px radius wherever possible.
- Rationale: Sharp corners signal precision and formality. Rounded corners feel human and gentle.
- DO: Buttons and cards 12px radius, inputs 8px radius, tooltips 6px
- DON'T: 0px radius anywhere, sharp right angles on modals or callouts

P4: 平面优先 (Flat Over Depth)
- Statement: Use color and spacing for hierarchy, not shadows or 3D effects. Glass effects are allowed for overlays only.
- Rationale: Drop shadows simulate physical objects — this is skeuomorphism. Flat design focuses on content.
- DO: Surface elevation via lighter backgrounds, glass (`backdrop-filter: blur`) on modals and dropdowns
- DON'T: Heavy drop shadows (blur >4px), gradient buttons simulating 3D, textures, inner shadows
</action>

### T003: Write Behavioral Layer Principles (P5-P6)

<action>
Write 2 behavioral layer principles in `.planning/design-philosophy.md`, after visual principles.

P5: 即时反馈 (Instant Feedback)
- Statement: Every interactive element must have hover, focus, and active states. No invisible buttons.
- Rationale: Users shouldn't guess what's clickable. States communicate: "I see you, I'm ready, you clicked me."
- DO: Buttons change color on hover, inputs show amber ring on focus, links underline on hover
- DON'T: Clickable elements with no hover state, focus outlines suppressed (`outline: none` without replacement), dead clicks with no visual response

P6: 平滑过渡 (Smooth Transitions)
- Statement: State changes should decelerate naturally — fast start, gentle arrival. 150-200ms.
- Rationale: Instant transitions feel jarring. Long transitions feel sluggish. Natural easing mimics physical motion.
- DO: `transition: all 150ms ease-out` for hover effects, `200ms ease-in-out` for page transitions
- DON'T: Instant state changes (0ms), bouncy easing (`cubic-bezier` with extreme values), transitions >300ms for UI feedback
</action>

### T004: Write Holistic Layer Principles (P7-P8) & Conflict Resolution

<action>
Write 2 holistic layer principles and a conflict resolution note in `.planning/design-philosophy.md`.

P7: 一致性优先 (Consistency First)
- Statement: The same element should look and behave the same across all pages. Reuse before creating.
- Rationale: Inconsistency erodes trust. Users learn patterns — break the pattern, break their confidence.
- DO: One button component with variants, not different buttons on different pages
- DON'T: Rounded corners of 8px here but 12px there, different hover colors for the same action

P8: 做减法 (Subtract Before Adding)
- Statement: Before adding any visual element, ask: "What if I removed it instead?" Uncertain? Remove it.
- Rationale: Every element competes for attention. Fewer elements = clearer message.
- DO: Check if whitespace alone can communicate the grouping before adding a border
- DON'T: Decorative elements (gradients, icons, dividers) that don't serve a functional purpose

Then add a short "原则冲突处理" section:
- When P5 (Instant Feedback) conflicts with P8 (Subtract): Feedback wins for interactive elements
- When P7 (Consistency) conflicts with P2 (Whitespace): Consistency in component behavior, whitespace adapts to context
</action>

4. Cold color accents (blue, purple, teal replacing amber/warm tones)
5. Information overload (dense layouts with no breathing room, long scroll pages)
6. (Bonus) Gradient buttons simulating 3D bevel/emboss
</action>
