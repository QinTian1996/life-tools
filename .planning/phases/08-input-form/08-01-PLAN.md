**State:**

- `gender: Gender` ('male' | 'female'), default 'male'
- `year`, `month`, `day`, `hour`, `minute` as string inputs (use useState)
- `shichen` string, default '子'
- `isLeapMonth: boolean`, default false (only relevant when calendar='lunar')
- `name: string`, default ''

**Validation function `isFormValid()`:**
- year: parseInt >= 1900 AND <= 2100
- month: parseInt >= 1 AND <= 12
- day: parseInt >= 1 AND <= 31
- gender: must be 'male' or 'female' (always valid since select)
- Return boolean

**Two-row layout (per D-10):**

Row 1 — horizontal flex, gap-4:
- Year input (number, 1900-2100, label "年份" for solar / "年份(农历)" for lunar)
- Month input (number, 1-12, label "月份" / "月份(农历)")
- Day input (number, 1-31, label "日期" / "日期(农历)")
- Gender select (男/女, label "性别")
- Button "排盘" (disabled={!isFormValid()}, variant="primary")

Row 2 — horizontal flex, gap-4, items-center:
- 公历/农历 toggle switch:
  - CSS toggle: relative inline-block w-11 h-6, rounded-full
  - Off (solar): bg-[var(--muted-foreground)]
  - On (lunar): bg-[var(--primary)]
  - Knob: absolute w-5 h-5 rounded-full bg-white top-0.5, transitions left position
  - When solar→lunar: show "闰月" checkbox next to month input
- Time mode toggle — three segments (精确时间 | 时辰 | 不知道):
  - Segmented control using flex with borders
  - Selected segment: bg-[var(--primary)] text-[var(--primary-foreground)]
  - Unselected: bg-transparent text-[var(--foreground)]

**Conditional time inputs:**
- If timeMode='precise': hour (0-23) + minute (0-59) inputs appear below Row 2
- If timeMode='shichen': select dropdown with 12 时辰 options (子丑寅卯...)
- If timeMode='unknown': no additional inputs

**Name field:**
- Below Row 2 and time inputs
- Input with placeholder "留空将自动生成" (per D-08)
- Full width or max-w-sm

**onSubmit handler:**
- Build BirthInput object with all fields
- If name is empty string, omit from object OR pass empty (D-09 says empty string passed)
- Call `onCalculate(input)`
- Note: For lunar calendar, the year/month/day are passed as lunar values; computeBazi in calculator.ts handles conversion via toSolar()

**Styling:**
- Use design tokens: bg-[var(--card)], border-[var(--border)], text-[var(--foreground)], text-[var(--muted-foreground)]
- Compact layout to minimize form height
- Inputs use h-10 px-3 rounded-[var(--radius-md)] border border-[var(--border)] bg-[var(--background)]

**Exports:**
