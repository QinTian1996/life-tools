## Clarifications

### Session 2026-05-09

- Q: 品牌导航栏是否需要适配暗色模式？ → A: 自动跟随系统暗色模式，品牌栏使用 CSS 变量自动切换色彩
- Q: LOGO 来源？ → A: 使用项目中已有的 `public/logo.svg`
- Q: 子页面标题显示位置？ → A: 品牌栏右侧，与 LOGO 同行
- Q: 首页点击 LOGO 的行为？ → A: 无操作，首页 LOGO 不响应点击（仅 hover 视觉反馈）

## User Scenarios & Testing *(mandatory)*

### User Story 1 - Brand Logo Navigation (Priority: P1)

作为一个访客，我在浏览任何一个子页面（比如聊天）时，左上角都能看到「喵十七」的品牌标识（LOGO + 文字），点击它能立刻回到首页。这样我随时能找回入口，不会迷路。

**Why this priority**: 这是最基础的用户体验 — 每个页面都应该有"回家"的路。没有这个，用户在子页面里没有导航上下文。

**Independent Test**: 打开任意子页面，看到左上角有品牌标识，点击后跳转到首页。

**Acceptance Scenarios**:

1. **Given** 用户在首页，**When** 页面加载完成，**Then** 左上角显示品牌标识（LOGO + "喵十七"文字），但点击不触发导航或刷新（已在首页），hover 时可有视觉反馈
2. **Given** 用户在任意子页面（如 /chat），**When** 页面加载完成，**Then** 左上角显示品牌标识（LOGO + "喵十七"文字），且标识可点击
3. **Given** 用户在子页面，**When** 点击左上角品牌标识，**Then** 立刻导航到首页（/），且页面正常渲染

---

### User Story 2 - Consistent Page Layout Structure (Priority: P1)

作为一个访客，无论我打开哪个子页面，它们都有一个统一的结构：顶部是品牌导航栏（带 LOGO），中间是内容区，整体视觉风格一致（同样的背景、颜色、字体感觉）。这样我觉得这个网站是"一个整体"，而不是东拼西凑的。

**Why this priority**: 视觉一致性是用户感知"品质"的核心。当前首页和聊天页风格不统一（首页无 header，聊天页有 inline header），需要统一。

**Independent Test**: 分别打开首页和聊天页，对比两者的整体结构 — 都应该有顶部品牌栏、一致的背景渐变、一致的色彩体系。

**Acceptance Scenarios**:

1. **Given** 用户打开首页（/），**When** 页面加载，**Then** 页面有顶部品牌栏，品牌栏包含 LOGO、文字、和回到首页的链接
2. **Given** 用户打开聊天页（/chat），**When** 页面加载，**Then** 页面有顶部品牌栏（取代原有的"聊天"标题栏），品牌栏样式与首页一致
3. **Given** 用户对比首页和子页面，**When** 观察整体视觉，**Then** 背景渐变色系、字体风格、间距节奏保持一致
4. **Given** 子页面原本有自己的页面标题（如聊天页的"聊天"），**When** 统一布局后，**Then** 页面标题显示在品牌栏右侧（与 LOGO 同行），用户知道自己在哪个功能里

---

### User Story 3 - Unified Design Language for Future Pages (Priority: P2)

作为这个项目的开发者/维护者，当我新增一个小工具页面时，不需要从头设计样式 — 只需要套用已有的页面框架（顶部品牌栏 + 内容区），就能自动获得统一的视觉风格。这样新页面和旧页面看起来像是"一家人"。

**Why this priority**: 这是长期可维护性的基础。没有统一的框架，每加一个页面都会引入新的样式偏差，越积累越乱。

**Independent Test**: 创建一个新的空白页面，套用框架后，它自动拥有品牌栏、正确背景、统一风格，无需额外写布局代码。

**Acceptance Scenarios**:

1. **Given** 开发者想添加新页面（如 /eat），**When** 创建页面文件，**Then** 可以复用统一的页面布局组件，自动获得品牌栏和背景
2. **Given** 统一布局被应用到新页面，**When** 页面渲染，**Then** 视觉风格（颜色、字体、间距）与现有页面一致
3. **Given** 统一布局支持显示当前页面标题，**When** 开发者传入页面标题，**Then** 标题显示在品牌栏上（如"今天吃什么"）

---

### Edge Cases

- 当用户在首页点击左上角品牌标识时，不会触发页面刷新或报错（已在当前页面）
- 品牌栏在移动端小屏幕上仍然可用 — LOGO 和文字不会被截断或溢出
- 品牌栏不应该遮挡或影响子页面的功能区域（如聊天输入框）
- 当页面内容很长需要滚动时，品牌栏应该固定在顶部（不随页面滚动消失）
- 当用户操作系统切换为暗色模式时，品牌栏色彩自动切换为暗色调，LOGO 和文字在暗色背景下仍然清晰可辨

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 系统必须在所有页面的左上角显示品牌标识，包含 LOGO 图形和"喵十七"文字
- **FR-002**: 品牌标识在子页面中必须是可点击的链接，点击后导航到首页（/）；在首页时不响应点击
- **FR-003**: 所有页面必须共享统一的顶部品牌导航栏组件
- **FR-004**: 品牌导航栏在子页面中应固定在顶部，不随页面内容滚动
- **FR-005**: 品牌导航栏右侧应显示当前子页面的标题（如"聊天"，与 LOGO 同行）
- **FR-006**: 所有页面的背景渐变色系必须统一为暖色调（当前 amber/orange 风格）
- **FR-007**: 页面布局框架必须可复用，新增页面时不需要重复编写布局代码
- **FR-008**: 现有聊天页的内联 header（"聊天"标题栏）必须迁移到统一品牌导航栏中
- **FR-009**: 首页现有内容（工具箱标题、描述、按钮列表、署名）在加入品牌栏后整体布局保持协调
- **FR-010**: 品牌导航栏的 LOGO 使用项目中已有的 `public/logo.svg`，搭配"喵十七"文字
- **FR-011**: 品牌导航栏必须自动适配系统暗色模式 — 使用 CSS 变量跟随全局亮/暗色彩切换，无需单独设计暗色版本

### Key Entities

- **品牌导航栏 (SiteHeader)**: 固定在页面顶部，包含 LOGO、品牌文字、可选的当前页面标题。LOGO 区域点击回到首页。
- **页面布局框架 (PageLayout)**: 包裹每个页面的外层容器，提供统一背景、品牌导航栏和内容区结构。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 用户从任意子页面点击品牌标识，在 1 秒内完成到首页的导航
- **SC-002**: 所有现有页面（首页 + 聊天页）应用统一框架后，视觉风格一致（背景色系、字体、品牌栏位置相同）
- **SC-003**: 新增页面时，使用布局框架可以在 5 分钟内完成页面骨架搭建（无需编写布局代码）
- **SC-004**: 品牌栏在常见屏幕尺寸（1280px - 1920px 宽）下均正常显示，LOGO 和文字不溢出
- **SC-005**: 聊天页迁移到统一框架后，原有聊天功能（发送消息、接收回复）完全不受影响

## Assumptions

- 当前 amber/orange 暖色系是用户认可的视觉风格，统一框架延续此风格
- 品牌导航栏是纯展示 + 导航功能，不涉及菜单、下拉等复杂交互
- 页面布局框架为所有子页面提供相同的结构，但子页面的内容区完全自定义
- 未来新增的页面（如 /bazi、/eat）也将使用统一框架，但本次不需要创建这些页面
- 移动端适配以"不破坏"为基本要求，不需要专门的移动端优化设计
- LOGO 文件已存在于 `public/logo.svg`，直接复用，无需重新设计
