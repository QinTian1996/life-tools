## References

## 使用流程

当用户提问涉及以下话题时，自动执行：

0. **如需系统化八字分析 → 加载 `bazi-analysis` 技能**（韦千里八步法），知识库古籍作为引用源
1. **判定话题所属分类** — 映射表见下方
2. **在对应目录下查找相关文件** — 用 `search_files` 搜索
3. **跨分类话题 → 优先去 原文/ 查找** — 因为原文可能覆盖多个分类
4. **读文件内容** — `read_file` 读取，按需只读关键段落
5. **基于内容回答** — 结合已注入的记忆给出答案

### 计划类问题的回答模式

当用户问"我的XX计划是什么"这类问题时，采用「已完成 ✅ / 待执行 ⏳」二分结构回答：

- **已完成**：列出已落实到位的项目，标注 ✓
- **待执行**：区分"有明确时间依赖"（如秋季）和"随时可做"两类
- **末尾**：给出最核心的 2-3 件优先待办，引导下一步

参考本次会话中对"风水布局的计划"的回答示例。

### 新增文件后的索引更新流程

用户说"知识库更新了"时：

1. `search_files` 扫描所有目录，识别新文件
2. 读取新文件头部，判断分类归属
3. 判断文件是否跨分类 → 放入 原文/ 或在分类目录间建交叉引用
4. 更新本 SKILL.md 的「已录入的内容」章节
5. 同步更新 `references/current-manifest.md`
6. 如内容重要，提炼关键事实到持久记忆
